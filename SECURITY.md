# Security Policy

## Supported Versions

Security fixes are applied to the latest released version.

## Reporting a Vulnerability

Please do not open public GitHub issues for security problems.

Report vulnerabilities privately to the maintainer through the security contact configured for the repository hosting this package. If no private security channel is configured yet, set up GitHub private vulnerability reporting before the first public release.

Include:

- a short description of the issue
- affected versions
- reproduction steps or a proof of concept
- impact assessment

You can expect an acknowledgement within a reasonable timeframe and coordinated disclosure after a fix is available.
